# Maestro Content Workflow

## Назначение

Этот этап ускоряет существующий процесс наполнения курса без новых разделов, сущностей и изменений Learning Engine.

Реализованы:

- сводка готовности курса на существующей странице конструктора;
- проверка видео внутри редактора урока;
- загрузка, просмотр, замена и безопасное удаление PDF и других материалов;
- облегченная загрузка структуры большого курса.

## Готовность курса

Блок «Готовность курса» расположен над деревом курса.

Урок считается готовым, если одновременно:

- опубликован;
- имеет `videoUrl`;
- имеет минимум один материал;
- имеет минимум одно активное домашнее задание.

Блок показывает:

- процент готовности;
- количество готовых и всех уроков;
- количество опубликованных уроков;
- черновики;
- уроки без видео;
- уроки без материалов;
- уроки без домашнего задания.

Карточки проблем кликабельны. После клика отображается список соответствующих уроков. Выбор урока открывает его в существующем конструкторе.

## Проверка видео

Редактор урока использует существующие:

- `parseVideoUrl`;
- `LessonVideoPlayer`.

После ввода ссылки CMS показывает:

- распознанного провайдера;
- структурную валидность ссылки;
- `videoId`;
- встроенный просмотр;
- действия «Открыть», «Заменить», «Удалить».

Поддерживаются:

- YouTube;
- Vimeo;
- Cloudflare Stream.

Проверка подтверждает формат и возможность построить embed-ссылку. Она не проверяет приватность видео, региональные ограничения или существование удаленного видео у провайдера.

## Работа с PDF и материалами

Материал урока показывает:

- название материала;
- тип;
- исходное имя локального файла;
- размер;
- дату загрузки;
- ссылку для открытия.

Доступные действия:

- открыть;
- заменить файл;
- скопировать ссылку;
- изменить порядок;
- удалить привязку.

При замене:

- загружается новый локальный файл;
- меняется только URL существующего материала;
- сохраняются название, тип, порядок и привязка к уроку.

Перед удалением привязки CMS показывает уроки, использующие тот же URL. Сам файл остается в медиатеке.

Перед удалением файла из медиатеки CMS также показывает все уроки, использующие этот файл, и предупреждает о неработающих ссылках.

### Исходные имена файлов

Для новых локальных загрузок рядом с файлом создается небольшой служебный metadata-файл с исходным именем и MIME-типом. Prisma schema не изменялась.

Существующие загрузки без metadata-файла продолжают работать. Для них CMS показывает текущее техническое имя файла.

## Производительность

Конструктор больше не загружает полные данные всех уроков курса.

`GET /admin/courses/:id/tree` возвращает только данные, необходимые для дерева и расчета готовности:

- идентификатор и название урока;
- порядок;
- публикацию;
- наличие видео;
- количество материалов;
- количество активных домашних заданий.

Полное описание, `videoUrl`, баллы и другие данные загружаются отдельным запросом только после выбора урока:

- `GET /admin/lessons/:id`

Материалы и домашнее задание также загружаются только для выбранного урока.

Метаданные локальных файлов определяются только для материалов выбранного урока. Медиатека читает файловые метаданные параллельно.

Такой подход рассчитан на рабочий сценарий курса из 40–100+ уроков без загрузки полного содержимого курса одним ответом.

## Используемые API

### Обновленные

- `GET /api/v1/admin/courses/:id/tree`
- `GET /api/v1/admin/materials?lessonId=:lessonId`
- `GET /api/v1/admin/media`
- `POST /api/v1/admin/media`
- `DELETE /api/v1/admin/media/:folder/:filename`

### Добавленные для существующих сценариев

- `GET /api/v1/admin/lessons/:id` — полные данные выбранного урока;
- `GET /api/v1/admin/materials/:id/usages` — места использования URL материала;
- `GET /api/v1/admin/media/:folder/:filename/usages` — места использования локального файла.

Новые endpoints используют существующую защиту `catalog.manage` и доступны только контентным администраторам.

## Рабочий сценарий

1. Открыть существующий курс в CMS.
2. Проверить блок «Готовность курса».
3. Нажать на проблемную категорию и открыть нужный урок.
4. В режиме редактирования вставить `videoUrl` и проверить встроенное видео.
5. В блоке материалов выбрать локальный PDF или указать готовый URL.
6. Проверить имя, размер, дату и открытие файла.
7. При обновлении PDF нажать «Заменить». Карточка материала и порядок сохранятся.
8. После наполнения проверить, что проблемные категории готовы к публикации.

## Ограничения

- Проверка видео не обращается к API YouTube, Vimeo или Cloudflare.
- После замены материала предыдущий локальный файл остается в медиатеке и может быть удален вручную после проверки использования.
- Старые локальные файлы без metadata-файла не имеют сохраненного исходного имени.
- Массовое редактирование, пакетная публикация и Drag & Drop намеренно не добавлялись.
